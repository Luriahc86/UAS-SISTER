🏗️ Arsitektur Sistem

                        

                            Pub-Sub Log Aggregator menggunakan arsitektur event-driven dengan Redis Stream sebagai broker internal
                            dan PostgreSQL untuk persistent storage. Deduplication dan concurrency control dilakukan di level database.

🔒
Idempotent Consumer
Event yang sama tidak diproses lebih dari 1x. Worker menggunakan INSERT ON CONFLICT DO NOTHING.

                        
🧬
Deduplication
Dedup key: (topic, event_id). PostgreSQL UNIQUE constraint menjamin tidak ada duplikat di database.

                        
⚡
Concurrency Control
Race condition dicegah oleh UNIQUE constraint + atomic upsert. Multi-worker bisa berjalan paralel.

                        
💾
Persistence
Docker named volume (pg_data, redis_data) memastikan data bertahan saat restart container.

                        
📊
ACID Transactions
Insert dedup + insert event + update stats dalam satu transaksi. Rollback jika ada kegagalan.

                        
🔄
Fault Tolerance
Redis PEL + consumer group memastikan event tidak hilang saat worker crash.
